# JetCheck 1.4 工程管理页面设计草案 v0.1

本文档定义尺寸工具桌面端应用中的 `工程管理` 页面。

这份文档只回答工程管理页面本身的问题，不展开单工程配置页面。

## 1. 页面定位

工程管理页面是尺寸工具桌面端应用的主入口窗口。

它负责：

1. 查看所有本地尺寸工程
2. 创建工程
3. 打开工程
4. 导入工程
5. 快速判断每个工程当前所处状态
6. 从工程管理窗口进入单工程配置窗口

它不负责：

1. 直接编辑标定、ROI、测量项、公差、验证内容
2. 承担单工程内的详细属性编辑
3. 替代工程配置窗口

## 2. 窗口角色

在 Windows 桌面端应用里，工程管理页面是独立窗口。

当前口径如下：

1. `工程管理` 和 `工程配置` 是两个不同窗口。
2. 用户先进入工程管理窗口，再打开某个工程的配置窗口。
3. 工程管理窗口始终面向“所有工程”。
4. 工程配置窗口始终面向“当前工程”。

这意味着工程管理窗口的核心心智不是“配置当前工程”，而是“管理工程集合”。

## 3. 页面目标

工程管理页面需要在 5 秒内回答下面几个问题：

1. 现在一共有多少个工程
2. 哪些工程是草稿
3. 哪些工程已经发布
4. 我要进入哪个工程继续配置

因此页面优先级是：

1. 先看全局
2. 再选工程
3. 最后进入配置

## 4. 页面结构

工程管理窗口采用三段结构：

1. 窗口标题栏
2. 工具栏
3. 工程列表区

## 5. 标题栏

标题栏遵循 Windows 应用窗口心智。

包含：

1. 应用图标
2. 窗口标题：`工程管理`
3. 窗口副标题：`所有尺寸工程`
4. 最小化、最大化、关闭按钮

标题栏不承载业务筛选或搜索。

## 6. 工具栏

工具栏放页面级操作，不放单工程步骤。

固定包含：

1. `新建工程`
2. `导入工程`
3. `刷新列表`

可选包含：

1. 搜索框
2. 排序方式
3. 只看草稿 / 只看已发布 过滤器

当前阶段工具栏不放：

1. 标定
2. ROI
3. 测量项
4. 公差
5. 验证
6. 发布

这些动作属于工程配置窗口，不属于工程管理窗口。

## 7. 工程列表区

工程列表区是页面主体。

它是工程管理页面最重要的区域。

## 8. 列表展示方式

工程列表区采用 Windows 应用常见的列表视图，而不是卡片墙。

默认采用：

1. 顶部表头
2. 行式列表
3. 单行选中
4. 当前选中行高亮

不采用：

1. 大面积卡片式平铺
2. 纯视觉化海报式入口
3. 工程信息碎片化散落

列表视图更适合承载：

1. 多工程对比
2. 按列排序
3. 快速扫描状态
4. 双击打开工程

## 9. 列表字段

工程列表至少展示下面这些字段：

1. 工程名称
2. 状态
3. 最近修改时间

字段说明如下：

### 9.1 工程名称

这是主字段，放最前面。

要求：

1. 可点击
2. 可双击打开工程配置窗口
3. 支持名称过长时省略

### 9.2 状态

状态只保留两种：

1. 草稿
2. 已发布

工程管理页面不展示更细的过程状态。

### 9.3 最近修改时间

用于快速定位最近工作的工程。

## 10. 行级交互

每一行工程支持下面几种交互：

1. 单击：选中当前工程
2. 双击：打开工程配置窗口
3. 行尾按钮 `进入配置`：打开工程配置窗口

当前阶段不在列表主行直接暴露太多操作按钮。

行内最多保留：

1. `进入配置`
2. 可选的更多菜单

更多菜单中可以放：

1. 重命名
2. 复制工程
3. 导出工程
4. 删除工程

这些动作不放成常驻主按钮，避免列表显得过于杂乱。

## 11. 排序与筛选

工程管理页面默认支持轻量排序和筛选。

默认排序：

1. 最近修改时间倒序

默认筛选：

1. 全部工程

当前阶段支持的筛选维度：

1. 状态

当前阶段不做复杂组合查询器。

## 12. 搜索

搜索框按工程名称做模糊匹配。

搜索结果只影响列表显示。

## 13. 新建工程

点击 `新建工程` 后，打开新建工程弹窗。

弹窗字段固定为：

1. 工程名称
2. 备注

提交后行为：

1. 在工程列表中生成新行
2. 默认状态为 `草稿`
3. 自动打开工程配置窗口

## 14. 导入工程

点击 `导入工程` 后，打开系统文件选择流程。

导入成功后：

1. 新工程进入工程列表
2. 保留导入工程原有状态信息
3. 默认不直接覆盖已有同名工程

同名冲突时：

1. 提示用户重命名
2. 或导入为副本

## 15. 打开工程配置窗口

工程管理页面进入工程配置页面的方式只有一类：

1. 打开单独的工程配置窗口

进入后：

1. 工程管理窗口仍保留在桌面上
2. 工程配置窗口显示当前工程
3. 配置窗口不再展示“所有工程列表”

这样两个窗口职责清楚：

1. 工程管理负责看全部
2. 工程配置负责改当前

## 16. 空状态

当本地还没有工程时，工程管理页面显示空状态。

空状态内容包括：

1. 说明文字：当前还没有尺寸工程
2. 主按钮：新建工程
3. 次按钮：导入工程

不显示空列表。

## 17. 异常状态

工程管理页面需要处理下面几类异常：

1. 工程列表加载失败
2. 导入失败
3. 工程文件损坏
4. 同名冲突

异常反馈原则：

1. 错误要落在当前操作附近
2. 明确告诉用户下一步怎么做
3. 不只给“失败”结果

## 18. 页面风格要求

工程管理页面的视觉风格遵循 Windows 桌面端应用心智。

要求如下：

1. 有明确窗口标题栏和窗口边界
2. 列表视图优先，不做网页式大卡片首页
3. 行选中、表头、排序、打开行为要像桌面软件
4. 页面重点放在工程集合管理，不放单工程步骤导航

## 19. 和工程配置页面的边界

工程管理页面只做到“选择工程并进入配置”。

下面内容全部归工程配置页面：

1. 标定
2. ROI
3. 测量项
4. 公差
5. 验证
6. 发布

如果工程管理页面把这些内容也带上，就会重新退回“一个大页面里塞所有事情”的 Web 心智。

## 20. 页面总结

工程管理页面是尺寸工具桌面端的“工程入口窗口”。

它的核心不是配置，而是：

1. 管理所有工程
2. 快速判断工程状态
3. 打开单工程配置窗口

页面设计必须始终围绕这一点，不和工程配置窗口混淆。
