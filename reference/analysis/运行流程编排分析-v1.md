# JetCheck 运行流程编排抽象建议 v1

## 1. 文档目的

在《项目变量清单与配置抽象建议 v1》基础上，本文聚焦“流程编排层”，将多项目中反复出现的运行差异统一成：

1. 流程节点标准模型
2. 状态机与中断机制
3. 外部设备联动接口规范
4. 六项目流程映射模板

目标是让“标准产品 + 项目定制”中的流程差异可配置、可复用、可追溯。

## 2. 流程编排总体模型

### 2.1 核心对象

1. `FlowTemplate`：项目级流程模板
2. `FlowRun`：单次检测实例
3. `FlowNode`：流程节点（等待触发、采图、检测、回传等）
4. `FlowEdge`：节点连接关系（成功、失败、超时、人工介入）

### 2.2 节点分类（v1）

| 节点类型 | 节点标识 | 作用 |
| --- | --- | --- |
| 触发节点 | `wait_trigger` | 等待 PLC/IO/按钮/文件事件 |
| 输入节点 | `acquire_image` | 采图或读入外部图片 |
| 处理节点 | `preprocess_roi` | ROI 裁剪、分区、模型定位 |
| 检测节点 | `run_detector` | 执行分类/检测/测量 |
| 判定节点 | `judge_result` | 业务规则汇总为 OK/NG |
| 联动节点 | `emit_signal` | 向气缸/机械臂/PLC 发中间动作信号 |
| 回传节点 | `report_result` | 回传 PLC/MES |
| 人工节点 | `wait_manual` | 人工确认/复检等待 |
| 收尾节点 | `finish` | 结束并归档记录 |

## 3. 状态机建议（任务级）

### 3.1 主状态

1. `IDLE`：空闲
2. `WAIT_TRIGGER`：等待触发
3. `RUNNING`：执行中
4. `WAIT_EXTERNAL_ACTION`：等待外部设备动作完成
5. `WAIT_MANUAL`：等待人工处理
6. `COMPLETED_OK`：完成且 OK
7. `COMPLETED_NG`：完成且 NG
8. `ABORTED_EXCEPTION`：异常中断

### 3.2 通用迁移规则

1. `WAIT_TRIGGER -> RUNNING`：收到有效触发信号
2. `RUNNING -> WAIT_EXTERNAL_ACTION`：当前节点需要设备动作反馈
3. `WAIT_EXTERNAL_ACTION -> RUNNING`：收到设备完成信号
4. `RUNNING -> WAIT_MANUAL`：触发告警或人工复检流程
5. `RUNNING -> COMPLETED_OK/COMPLETED_NG`：判定完成并收尾
6. 任意执行态 -> `ABORTED_EXCEPTION`：超时/漏图/模型异常等不可恢复异常

## 4. 时间控制模型

### 4.1 三层超时

1. `node_timeout_s`：单节点超时（如单次检测）
2. `stage_timeout_s`：阶段超时（如一面 2 张图）
3. `task_timeout_s`：整任务超时（如 10s）

### 4.2 超时动作策略

| 策略 | 含义 | 典型项目 |
| --- | --- | --- |
| `set_ng_and_alarm` | 置 NG + 告警 + 等待人工 | 天阁压壳、天阁背板 |
| `retry_once_then_ng` | 自动重试一次，失败置 NG | 可选 |
| `abort_and_hold` | 直接中断并保持工位 | 可选 |

## 5. 异常分类与中断机制

### 5.1 异常码建议

| 异常码 | 类别 | 默认动作 |
| --- | --- | --- |
| `E_TIMEOUT` | 超时 | `set_ng_and_alarm` |
| `E_CAPTURE_MISSING` | 漏图 | `set_ng_and_alarm` |
| `E_MODEL_RUNTIME` | 模型异常 | `set_ng_and_alarm` |
| `E_SIGNAL_LOST` | 信号中断 | `abort_and_hold` |
| `E_STORAGE_LOW` | 存储不足 | `alarm_and_block` |
| `E_NETWORK` | 对接失败 | `alarm_and_retry` |

### 5.2 中断优先级

1. 安全相关中断（设备/工位异常）优先级最高
2. 超时中断高于普通业务 NG
3. 业务 NG 不等于流程异常，应能正常收尾并回传

## 6. 外部联动接口建议

### 6.1 输入信号（示意）

| 信号 | 方向 | 含义 |
| --- | --- | --- |
| `start_detect` | 外部 -> JetCheck | 触发开始 |
| `stage_done_ack` | 外部 -> JetCheck | 中间动作完成反馈 |
| `recheck_confirm` | 人工/外部 -> JetCheck | 允许复检 |

### 6.2 输出信号（示意）

| 信号 | 方向 | 含义 |
| --- | --- | --- |
| `half_done_rotate` | JetCheck -> 气缸/IO | 半面完成，允许旋转 |
| `detect_ok` | JetCheck -> PLC | 检测通过 |
| `detect_ng` | JetCheck -> PLC | 检测失败 |
| `detect_exception` | JetCheck -> PLC/MES | 异常中断 |
| `record_sync` | JetCheck -> MES | 记录同步 |

## 7. 流程模板（伪 DSL）

```yaml
flow_id: tg_backplate_v1
task_timeout_s: 10
nodes:
  - id: n1
    type: wait_trigger
    config: { source: io, signal: pedal_down }
  - id: n2
    type: acquire_image
    config: { image_slot: A_top }
  - id: n3
    type: run_detector
    config: { item: defect_check }
  - id: n4
    type: emit_signal
    config: { signal: half_done_rotate }
  - id: n5
    type: wait_trigger
    config: { source: io, signal: rotate_done }
  - id: n6
    type: acquire_image
    config: { image_slot: A_bottom }
  - id: n7
    type: judge_result
    config: { rule: any_ng_is_ng }
  - id: n8
    type: report_result
    config: { targets: [mes] }
  - id: n9
    type: finish
edges:
  - from: n1
    to: n2
    on: success
  - from: n2
    to: n3
    on: success
  - from: n3
    to: n4
    on: success
  - from: n4
    to: n5
    on: success
  - from: n5
    to: n6
    on: success
  - from: n6
    to: n7
    on: success
  - from: n7
    to: n8
    on: success
  - from: n8
    to: n9
    on: success
on_exception:
  action: set_ng_and_alarm
  goto: wait_manual
```

## 8. 六项目流程映射要点

### 8.1 天阁压壳螺纹有无攻丝

1. 前置约束：气密性 NG 可直接跳过视觉
2. 触发：PLC
3. 检测：单图单项，计数型判定
4. 回传：PLC + MES
5. 关键约束：10s 超时直接 NG + 告警 + 人工

### 8.2 天阁背板缺陷

1. 触发：踏板 IO
2. 中间联动：半面完成后发信号驱动旋转气缸
3. 图像：4 图任务
4. 回传：MES（含复检最新策略）
5. 关键约束：复检保留本地全量记录

### 8.3 马斯特 X 光加热丝

1. 触发：外部图片放置/接口逐张触发
2. 链路：定位模型 -> 缺陷模型串联
3. 人工参与：人工看最终结果
4. 混料：单图多零件、可能多型号并存
5. 输出：结果图建议叠加统计

### 8.4 三洋安全座椅零部件

1. 前置：批次号输入 + 自动序号
2. 触发：实体按钮
3. 链路：卡扣方向 + 螺杆长度并行
4. 模板包：左右件双模板判定（含不匹配）
5. 输出：结果 + 测量值 + 追溯字段导出

### 8.5 血氧纱布缺陷

1. 触发：实体按钮
2. 联动：桁架 8 次移动采图（单面）
3. 双轮：翻面后重复流程（总 16 图）
4. 复检：人工处理缺陷后可复检
5. 可视化：ROI 区域 + 缺陷点位

### 8.6 瑾辰汽车饰件

1. 触发：实体按钮
2. 联动：机械臂带相机移动后第二次采图
3. 检测项：卡扣 + 焊点并行
4. ROI：手绘多区域
5. 判定：任一 ROI NG -> 图 NG -> 总 NG

## 9. 第一版流程引擎能力边界（建议）

### 9.1 v1 必须支持

1. 顺序流（串行节点）
2. 条件分支（成功/失败/超时）
3. 多图循环节点
4. 中间联动信号节点（emit + wait ack）
5. 超时统一处理与异常码归档
6. 人工复检回路（可选）

### 9.2 v1 可暂缓

1. 任意并行网关（复杂并发同步）
2. 可视化拖拽式流程编排器
3. 跨任务依赖编排

## 10. 与配置层的接口约定

### 10.1 FlowTemplate 引用配置

1. `input`：决定采图节点参数
2. `pipeline`：决定检测节点模型和顺序
3. `judge`：决定判定节点规则
4. `exception`：决定中断动作
5. `integration`：决定回传节点目标

### 10.2 运行记录最小字段

1. `flow_run_id`
2. `project_id`
3. `node_id`
4. `node_status`
5. `start_time` / `end_time`
6. `exception_code`（如有）
7. `result_snapshot`
