# JetCheck 项目变量清单与配置抽象建议 v1

## 1. 文档目的

基于已完成的 6 个试点项目需求梳理，本文将“项目差异”进一步下沉为可配置变量，形成：

1. 统一变量字典（做什么、怎么配）
2. 配置模型草案（建议的数据结构）
3. 六个项目的映射示例（怎么把需求落到配置）
4. 第一版实施优先级（先做哪些配置能力）

本文定位为“产品/研发/算法共同评审输入”，不是最终技术实现说明。

## 2. 配置抽象总体原则

### 2.1 分层原则

项目差异按三层承载：

1. 项目配置层（大多数需求通过配置解决）
2. 流程编排层（多步触发、设备联动、复检回路）
3. 标准底座层（采图、处理、检测、记录、展示基础能力）

### 2.2 表达原则

1. 算法输出与业务判定分离
2. 检测流程与页面渲染分离
3. 对接协议与业务字段映射分离
4. 异常和超时作为一等配置，不写死在代码分支里

## 4. 规则表达建议（Judge DSL v1）

建议把业务判定规则做成可配置表达式，避免每个项目写分支代码。

示例 1：天阁压壳（计数型）

```json
{
  "when": "task_complete",
  "if": "count(label=='有攻丝')==2 && count(label=='无攻丝')==0",
  "then": "OK",
  "else": "NG"
}
```

示例 2：背板缺陷（任意缺陷即 NG）

```json
{
  "when": "task_complete",
  "if": "exists(defect==true)",
  "then": "NG",
  "else": "OK"
}
```

示例 3：多检测项汇总（三洋/瑾辰）

```json
{
  "when": "task_complete",
  "if": "any(item_result=='NG')",
  "then": "NG",
  "else": "OK"
}
```

## 5. 六项目配置映射（v1）

### 5.1 天阁压壳螺纹有无攻丝

- `trigger.source=plc`
- `result.targets=["plc","mes"]`
- `input.task_image_count=1`
- `roi.mode=manual`（两孔区域）
- `pipeline.type=single`
- `judge.confidence.enabled=false`
- `judge.rule_set=计数型规则（2有攻丝且0无攻丝）`
- `exception.timeout=10s -> NG + 告警 + 等待人工`
- `trace.fields.mes=产品码/检测时间/原图/视觉结果`

### 5.2 天阁背板缺陷

- `trigger.source=io`（踏板）
- `flow.intermediate_signal=half_done_to_cylinder`
- `input.task_image_count=4`（正反面*上下半面）
- `roi.mode=full`
- `pipeline.type=single`
- `judge.rule_set=4图任一NG则任务NG`
- `judge.confidence.enabled=true`（阈值待定）
- `recheck.enabled=true`
- `recheck.record_policy=keep_all`
- `recheck.mes_sync_policy=latest_only`

### 5.3 马斯特 X 光加热丝

- `trigger.source=file_drop` 或 `api`
- `input.mode=external_image`
- `input.task_image_count=1`
- `input.mixed_product=true`
- `roi.mode=model`
- `pipeline.type=cascade`（零件定位 -> 缺陷检测）
- `judge.rule_set=图上任一零件NG则整图NG`
- `ui.runtime.show_statistics=true`（可展示 OK/NG 数量）

### 5.4 三洋安全座椅零部件

- `trigger.source=button`
- `input.camera_count=2`，`input.task_image_count=2`
- `pipeline.type=parallel`（卡扣方向 + 螺杆测量）
- `pipeline.item[].match_policy=allow_mismatch`（左右模板包并行）
- `judge.rule_set=卡扣NG或螺杆NG即总NG`
- `judge.tolerance.profile_ref=按型号读取`
- `trace.id_strategy=batch_plus_seq`
- `trace.fields.mes=批次号/产品编号/检测时间/检测结果`

### 5.5 血氧纱布缺陷

- `trigger.source=button`
- `input.task_image_count=16`（正反面*8区）
- `roi.mode=grid`
- `roi.grid_spec=4x4（待现场确认）`
- `pipeline.type=single`
- `judge.rule_set=任一子图/任一图NG即任务NG`
- `judge.confidence.enabled=true`（阈值待定）
- `ui.runtime.show_roi_index=true` + `show_ng_location=true`
- `recheck.enabled=true`（人工处理后复检）

### 5.6 瑾辰汽车饰件

- `trigger.source=button`
- `input.task_image_count=2`（机械臂移位后再次拍摄）
- `roi.mode=manual`（卡扣区/焊点区）
- `pipeline.type=parallel`（卡扣检测 + 焊点检测）
- `judge.rule_set=图上任一区域NG则图NG，任一图NG则任务NG`
- `ui.runtime.show_ng_location=true`（显示 ROI 位置）

## 7. 第一版实施优先级（建议）

### 7.1 必须先做（MVP）

1. 可配置触发源（PLC/IO/按钮/外部图片）
2. 多图任务组织（N 张图和顺序定义）
3. ROI 四模式（full/manual/model/grid）
4. 业务判定规则层（任意NG、计数型、多检测项汇总）
5. 统一异常策略（尤其 10s 超时 -> NG+告警）
6. MES 字段映射与回传（最小字段集）
7. 运行页 NG 位置可视化

### 7.2 可以后做（v1.5+）

1. 复杂统计叠加（按型号/类别数量）
2. 规则可视化编辑器（非代码 DSL 编辑）
3. 自动重试高级策略
4. 多系统并行回传的可靠队列机制

## 8. 当前仍需补齐的配置关键项

1. 各项目触发点位/回传点位命名与状态码枚举
2. 图像序列与面别编号规范（尤其 4 图/16 图项目）
3. 置信度阈值适用项目与初始推荐值
4. 复检在本地与 MES 的字段映射细则
5. 马斯特项目统计叠加最终字段清单
6. 三洋项目左右模板包“不匹配”的业务归类（NG/异常/忽略）
