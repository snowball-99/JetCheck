# 马斯特X光加热丝缺陷检测需求与场景梳理

## 1. 文档信息

- 项目名称：马斯特 X 光加热丝缺陷检测
- 客户/产线/工位：马斯特
- 当前阶段：需求与场景梳理
- 说明：本文结合当前收集到的项目信息做项目化梳理
- 目的：在标准产品能力之上，先明确该项目的输入方式、批量处理场景、多模型链路、判定规则、可视化诉求与异常处理边界，为后续方案设计与定制项拆分提供基础

## 2. 项目背景与目标

### 2.1 当前背景

- 该项目使用 X 光图像做缺陷检测。
- 图像不是相机实时采集，而是由 X 光仪器拍摄后，通过指定位置/接口传入系统。
- 一张图中会同时出现多个零件。
- 现场存在混料检测场景，即同一批图中可能包含多种产品。

### 2.2 检测目标

- 基于现有示例项目基线，当前已知的业务目标是：
  - 检测零件上的气泡、熔断等缺陷。
- 当前用户补充口径：
  - 采用“先识别零件位置，再基于零件区域检测缺陷”的链路。

### 2.3 业务目标

- 从 X 光图像中识别待检零件位置。
- 针对零件区域执行缺陷检测。
- 对单张图中的多个零件进行汇总判定。
- 由操作员查看最终结果。

### 2.4 当前已明确的目标口径

- 拦截目标：
  - 当前可理解为存在气泡、熔断等缺陷的零件。
- 成功标准：
  - 当前尚未给出量化验收指标。
  - 现阶段可先按“能够稳定处理接口输入的 X 光图像、支持多零件检测、能给出最终结果并标出缺陷位置”作为试点阶段目标。

## 3. 现场角色与业务场景

### 3.1 使用角色

- 现场使用者：质检操作员。

### 3.2 触发时机

- 当前未给出严格的现场触发时机定义。
- 结合当前描述，初步理解为：
  - X 光图像已产生并放到指定位置。
  - 系统按逐张方式读取并输出检测结果。
  - 操作员主要查看最终结果。

### 3.3 批量规模

- 当前口径：一批几百张。

## 4. 现场业务流程

根据当前输入，现场主流程初步梳理如下：

1. X 光仪器生成图像。
2. 图像被放到指定位置，并通过接口输入系统。
3. 系统按逐张方式执行检测。
4. 先基于第一模型识别图中零件位置，生成后续检测所需 ROI。
5. 再基于第二模型，对识别出的具体零件区域做缺陷检测。
6. 系统输出当前图像的检测结果。
7. 操作员查看最终结果。

### 4.1 当前流程理解

- 当前更像“文件/接口输入 + 批量逐张处理”的模式，而不是单件工位实时节拍模式。
- 当前未体现设备级联动控制，重点在图像消费、检测执行与结果展示。

## 5. 输入与采图场景

## 6. 检测链路理解

### 6.1 标准产品能力映射

结合现有 JetCheck 标准产品能力，当前项目可初步映射为：

1. 图像获取
   - 接口输入单张 X 光图像。
2. 图像处理
   - 使用第一模型做模型 ROI，识别零件位置。
3. 图像检测
   - 使用第二模型，对零件 ROI 执行缺陷检测。

### 6.2 当前链路特征

- 当前项目不需要手绘 ROI。
- 这里的 ROI 不是人工画框，而是由第一模型自动识别零件位置生成。
- 这是典型的“模型 ROI + 缺陷检测”串联链路。
- 单张图中有多个零件，因此会产生多个子结果。

### 6.3 与示例基线的一致点

- 当前项目和示例基线中的“马斯特 X 光”能力基本一致：
  - 单张图像接口输入
  - 使用“加热丝零件识别模型”做模型 ROI
  - 在指定零件类型上进行缺陷检测

### 6.4 当前尚未明确的链路细节

- 第一模型输出的零件类别有哪些。
- 第二模型究竟对哪些类别执行缺陷检测。
- 混料场景下是否需要按不同产品类型分别统计结果。
- 单张图内若存在多个产品类型，是否全部参与缺陷检测。

## 7. 结果判定规则

### 7.1 子结果判定

- 当前口径：
  - 有缺陷就是 NG。
  - 无缺陷就是 OK。

### 7.2 单图判定

- 单张图中只要有 1 个零件为 NG，则该图为 NG。

### 7.3 总结果判定

- 因为当前一次处理 1 张图，所以总结果与单图结果一致。

### 7.4 阈值规则

- 当前理解：
  - 可能需要支持置信度阈值调节。
- 当前尚未明确：
  - 是否按统一阈值处理所有缺陷类型。
  - 阈值是否由项目级配置控制。
  - 阈值调整是否需要现场可配。

### 7.5 当前尚未明确的判定细节

- 单图中多个零件同时 NG 时，是否需要进一步区分严重程度。
- 若模型输出低于阈值的疑似缺陷，按 OK、NG 还是人工确认处理。
- 若第一模型漏识别零件，单图结果按 NG 还是按异常。
- 混料场景下，不同类别零件的 OK/NG 是否需要分别统计后再给总图结果。

## 11. 相对标准产品的潜在定制点

基于当前输入，已初步识别出以下可能的项目定制点：

1. 接口图像输入定制
   - 图像来自指定位置/接口，而不是相机实时采集。
2. 批量逐张处理定制
   - 现场按一批几百张图的方式逐张检测。
3. 多模型串联定制
   - 先做零件识别，再做缺陷检测。
4. 混料检测定制
   - 同一批图中可能存在多种产品。
5. 业务判定逻辑定制
   - 单图中任一零件 NG，则整图 NG。
6. 阈值配置定制
   - 可能需要支持置信度阈值调节。
7. 结果可视化定制
   - NG 时展示具体缺陷位置。
   - 可能需要展示 OK/NG 数量与型号统计信息。
