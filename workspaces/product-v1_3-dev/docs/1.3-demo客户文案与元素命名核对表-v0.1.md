# JetCheck 1.3 客户文案与元素命名核对表 v0.2

## 范围说明

- 本表按当前 `workspaces/product-v1_3-dev/demo/` 中正式系统会保留的客户可见界面梳理。
- 覆盖范围：客户端、平台端、运行态、记录详情、弹窗、toast、banner、空状态、示例数据命名。
- 不含纯开发内部变量名、DOM id、状态机字段名。
- 表中“所在位置”优先写客户实际看到的位置和触发条件。
- 已按正式系统口径忽略所有 Demo 调试入口、模拟按钮、内部评审入口；入口页已移除，不再纳入核对。
- 表中“建议表达”按“工厂现场用户第一次看到也能理解”的口径给出，优先避免研发术语、系统实现术语。

## 总体结论

| 维度 | 现状判断 | 建议方向 |
| --- | --- | --- |
| 页面基调 | 仍有部分内部说明和技术验证口吻 | 改成正式产品口吻，只保留与操作直接相关的信息 |
| 命名方式 | 同时混用“工具/实例/ROI/模型/业务结果/运行模式”等多套语言 | 统一改成“对象名 + 当前动作 + 结果” |
| 异常提示 | 多数能看懂，但部分偏系统内部，如“阻断阈值”“配置失效”“回填” | 改成“为什么不能做 + 现在该做什么” |
| 示例数据 | 带项目代号、技术口径、ROI 表达、模版表达 | 改成更贴近产线对象与检测目的的命名 |
| 平台端文案 | 有未承接导航和不完整入口 | 收口为当前已实现能力，避免“看起来能点但不能用” |

## 优先级建议

| 优先级 | 建议先改的部分 | 原因 |
| --- | --- | --- |
| P0 | 登录、解绑、运行失败、空间不足、删除限制等异常提示 | 这些直接影响客户能否理解“为什么不能做” |
| P0 | 检测工具配置页里的“实例 / ROI / 分区模型 / 输入目标”等术语 | 这是客户高频操作区 |
| P1 | 运行态、记录详情、导出弹窗中的“子图 / 结果渲染图 / 业务结果” | 影响结果理解和导出操作 |
| P1 | 平台端“客户端配额 / 硬件识别码 / 未承接导航” | 影响管理端可信度 |
| P2 | 示例项目名、模型名、参数字段中文化 | 影响成品感，但不阻断理解 |
