# JetCheck 版本设计文档 v1（评审版）

## 1. 文档信息

- 文档名称：JetCheck 版本设计文档
- 文档版本：v1.0
- 编写日期：2026-03-24
- 适用范围：JetCheck 平台端、客户端
- 适用对象：产品、开发、测试
- 评审基线：以最新版 demo 为准

本文档用于从 `需求说明`、`方案描述`、`详细设计` 三个维度，对本期版本设计进行统一说明，供开发和测试评审使用。

## 2. 本期设计范围

本期共包含六个需求模块：

1. 多客户端管理
2. 检测工具模块改造
3. 检测记录改造
4. 模型管理
5. 相机管理
6. 存储管理

## 3. 设计原则

1. 以当前 demo 已确认的页面结构和交互口径为准，不再沿用已被替换的旧方案。
2. 客户端与平台端的身份关系以“手机号账号 + 设备自动绑定”为核心。
3. 资源管理类能力尽量独立成模块，避免与业务配置动作混用。
4. 高风险操作必须显式提醒并二次确认。
5. 运行态优先展示结果，配置态优先展示结构关系。
6. 测试评审以当前范围内可落地、可验证的功能为准，不将占位模块纳入完整验收。

## 4. 多客户端管理

### 4.1 需求说明

当前系统需要建立平台端与客户端之间稳定、可追踪的客户端身份体系，并将客户端数量控制绑定到企业账号维度。用户使用手机号账号登录客户端后，系统需要自动识别当前设备，并在额度范围内完成绑定。平台端需要能够查看当前账号下已绑定客户端的使用情况和在线状态。

本模块主要解决以下问题：

1. 客户端身份无法稳定映射到平台端账号体系。
2. 客户端额度缺少统一控制口径。
3. 平台端无法准确展示“当前账号下有哪些客户端正在使用”。

### 4.2 方案描述

本期采用“账号登录 + 设备自动绑定”的方案：

1. 客户端登录页采用手机号 + 登录密码登录。
2. 登录成功后自动读取当前设备硬件识别码。
3. 若当前设备已绑定至当前企业账号，则直接进入系统。
4. 若当前设备未绑定，则在额度充足时自动完成绑定并进入系统。
5. 若额度不足，则登录失败并提示无法绑定新设备。
6. 客户端提供本地解绑入口，解绑后当前设备回到未绑定状态。
7. 平台端不提供解绑入口，只展示当前账号下已绑定客户端列表。

平台端页面收口为“客户端管理 + 账号中心”：

1. 客户端管理页展示当前账号配额及已绑定客户端列表。
2. 账号中心用于维护手机号、联系人、登录密码、企业名称。

### 4.3 详细设计

#### 4.3.1 客户端登录页

页面字段：

1. 客户端名称
2. 手机号
3. 登录密码
4. 登录按钮
5. 错误提示区
6. Demo 场景按钮

交互规则：

1. 登录页统一按“首次绑定/未绑定”态展示。
2. 不再为“已绑定”“解绑后重登”等状态单独设计页面。
3. 点击登录时执行以下校验：
   - 客户端名称不能为空
   - 手机号格式合法
   - 登录密码不能为空
   - 当前是否在线
   - 当前账号额度是否已满
4. 登录成功后自动写入客户端名称并完成绑定。
5. 已绑定设备后续进入客户端时，直接进入系统，不再回到登录页。

#### 4.3.2 客户端解绑

入口位置：

1. 设置 > 客户端信息

操作流程：

1. 用户点击“解绑当前客户端”
2. 弹出二次确认
3. 确认后解除当前设备绑定关系
4. 释放对应客户端额度
5. 页面返回登录入口

#### 4.3.3 平台端客户端管理

页面结构：

1. 页头展示：`客户端配额：已用/总量`
2. 筛选区：
   - 在线状态
   - 关键词（客户端名称、硬件识别码）
3. 列表字段：
   - 客户端名称
   - 硬件识别码
   - 绑定时间
   - 在线状态
   - 最近离线时间

当前明确不包含：

1. 平台端解绑
2. 未绑定状态展示
3. 最近登录时间
4. 最近心跳时间

#### 4.3.4 平台端账号中心

字段：

1. 登录手机号
2. 联系人
3. 登录密码
4. 企业名称

规则：

1. 手机号必须为 11 位。
2. 所有字段必填。
3. 保存后立即刷新共享数据。

## 5. 检测工具模块改造

### 5.1 需求说明

现有检测工具结构无法支撑更复杂的图像获取、图像处理和模型推理关系，运行页也无法反映真实业务层级。需要对检测工具从配置到运行进行整体重构，使其能够支持：

1. 多图像获取来源
2. 图像级与子图级结果表达
3. 更清晰的配置结构
4. 与外部系统结果汇总逻辑的协同

### 5.2 方案描述

本期将检测工具拆分为“配置态”和“运行态”两种使用方式。

配置态按三步组织：

1. 图像获取
2. 图像处理
3. 图像检测

运行态按三层结果组织：

1. 总结果
2. 每张图结果
3. 每张子图结果

其中：

1. 图像获取负责定义原始输入来源和示例图。
2. 图像处理负责基于示例图定义 ROI 或分类处理逻辑。
3. 图像检测负责将多个图像处理结果与本地模型组合成可运行检测实例。
4. 正式检测触发由外部系统控制，demo 中仅保留演示触发按钮。

### 5.3 详细设计

#### 5.3.1 工具总览

页面内容：

1. 工具卡片列表
2. 新建工具入口

交互规则：

1. 点击卡片空白区域进入运行页。
2. 点击卡片中的“编辑”进入配置页。
3. 卡片不展示复杂搭建说明与统计块。

#### 5.3.2 工具配置页

页面结构：

1. 页头：工具名称 + 编辑名称
2. 步骤导航：
   - 图像获取
   - 图像处理
   - 图像检测
3. 主体：当前步骤内容
4. 底部：上一步、下一步、完成并返回

规则：

1. 三类实例安全上限统一为 20。
2. 页面显示格式为 `当前数/20`，位于新增按钮左侧。

#### 5.3.3 图像获取实例

支持类型：

1. 相机获取
2. 接口获取

字段：

1. 实例名称
2. 获取类型
3. 相机或接口来源
4. 示例图片

规则：

1. 创建时必须上传示例图片。
2. 示例图会用于后续 ROI 绘制及运行态底图展示。

#### 5.3.4 图像处理实例

支持模式：

1. ROI 绘制
2. 目标检测分类

ROI 绘制规则：

1. 基于某个图像获取实例的示例图进行绘制。
2. 支持多个 ROI 区域。
3. 支持多个不检测区域。
4. 支持快捷生成：
   - 全图
   - 4 等分
   - 9 等分
   - 16 等分

目标检测分类规则：

1. 必须选择本地 `分类` 模型。
2. 不再保留处理说明字段。

#### 5.3.5 图像检测实例

字段：

1. 检测实例名称
2. 关联图像处理实例
3. 使用模型

规则：

1. 图像处理实例按图像获取实例分组展示。
2. 同一个检测实例可多选多个图像处理实例项。
3. 检测模型从本地模型中选择。
4. 图像处理项采用紧凑列表展示，不显示多余文件名和冗余说明。

#### 5.3.6 运行页

页面结构：

1. 左侧：
   - 每张图结果
   - 相机信息
2. 中间：
   - 当前图像推理结果
3. 右侧：
   - 总结果
   - 标签信息

运行规则：

1. 页面不提供正式“启动检测”按钮。
2. demo 中使用 `Demo: 模拟信号` 演示。
3. 如需回到演示基线，使用 `Demo: 重置数据`。
4. 检测执行后按图像获取实例顺序自动展示每张图结果。
5. 全部图像展示完成后输出最终总结果。

#### 5.3.7 推理结果展示

显示逻辑：

1. 中间主区域的底图为当前图像获取实例原图。
2. 子图结果按 ROI 坐标映射回原图。
3. 每个 ROI 仅显示：
   - ROI 序号
   - ROI 业务结果 `OK/NG`
4. ROI 内部的检测结果框展示：
   - 类别
   - 置信度
5. 图像下方不再显示图像名称。

#### 5.3.8 相机信息与标签

相机信息：

1. 位于左侧下方
2. 若来源为相机，则显示：
   - 相机名称
   - 参数组
   - 查看相机画面
3. 若来源为接口，则提示当前为接口输入，不显示相机信息

标签信息：

1. 位于右侧
2. 支持给当前检测记录添加最多 3 个标签
3. 以 `已添加 x/3` 形式展示

#### 5.3.9 相机画面弹窗

结构：

1. 左侧：相机画面
2. 右侧：参数信息

状态口径：

1. 已连接
2. 异常

## 6. 检测记录改造

### 6.1 需求说明

原检测记录结构只有两层，无法匹配当前检测工具的三层结果结构，也无法清晰支撑图像级和子图级查看与导出。需要将检测记录重构为与运行页一致的层级模型，并重新设计详情和导出方式。

### 6.2 方案描述

本期将检测记录调整为三层数据结构：

1. 总记录
2. 图像记录
3. 子图记录

页面采用“两级列表 + 结果查看弹层”的方案：

1. 一级页面展示总记录列表。
2. 点击“查看详情”后展示该总记录下的图像记录列表。
3. 图像记录可进一步查看结果图或查看子图结果。
4. 结果图查看交互与检测运行页尽量复用。

### 6.3 详细设计

#### 6.3.1 一级列表

字段：

1. 记录编号
2. 工具名称
3. 触发时间
4. 业务结果
5. 标签
6. 操作（查看详情）

筛选条件：

1. 工具
2. 业务结果
3. 时间范围
4. 关键词（记录编号、工具名称）

分页：

1. 一级列表支持分页
2. 当前每页 10 条

#### 6.3.2 二级详情

展现形式：

1. 弹窗展示

字段：

1. 图像获取实例
2. 输入来源
3. 图像业务结果
4. 子图数量
5. 操作
   - 查看结果图
   - 查看子图

#### 6.3.3 查看结果图

设计目标：

1. 复用运行页图像结果查看逻辑
2. 保持图像级和子图级查看的一致性

交互规则：

1. 左侧为主查看区，展示原图与 ROI
2. 右侧为子图缩略区
3. 支持控制 ROI 图层显隐
4. 点击子图缩略图可聚焦查看
5. 双击 ROI 可直接切到对应子图

#### 6.3.4 查看子图

规则：

1. 子图名称采用 `图像处理实例名称 + ROI序号`
2. 不再额外显示“当前图像结果”摘要
3. 子图展示结构与运行页子图结果查看保持一致

#### 6.3.5 子图附加动作

1. 支持标记可疑
2. 支持取消标记

#### 6.3.6 导出设计

导出对象：

1. 当前筛选范围内的检测记录

导出分组：

1. 检测记录
2. 图像记录
3. 子图记录

每组导出项：

1. 检测记录：
   - 检测记录 Excel 文档
2. 图像记录：
   - 原图
   - 结果渲染图
   - 图像记录 Excel 文档
3. 子图记录：
   - 原图
   - 结果渲染图
   - 子图记录 Excel 文档

弹窗规则：

1. 顶部仅提示“当前筛选范围内的检测记录”
2. 选项以横向勾选项方式展示
3. 不展示长段解释性文案

## 7. 模型管理

### 7.1 需求说明

现有模型能力如果完全依附于检测实例配置，会导致“模型管理”和“模型选择”心智混杂。需要建立独立模型管理模块，并明确本地模型与云端模型的职责边界。

### 7.2 方案描述

本期采用“本地模型为主，云端模型为添加方式”的设计：

1. 客户端新增一级页面 `模型管理`
2. 页面主体展示本地模型列表
3. 云端模型通过抽屉方式打开，仅用于查询和下载到本地
4. 检测实例中选择模型时，使用轻量本地模型选择器

本地模型来源统一收敛为两类：

1. 云端同步
2. 本地上传

### 7.3 详细设计

#### 7.3.1 本地模型管理页

页头操作：

1. 从云端添加
2. 本地上传

查询区：

1. 模型名称或版本编号
2. 场景类型

列表字段：

1. 模型名称
2. 场景类型
3. 版本编号
4. 来源
5. 添加时间
6. 引用情况
7. 操作

规则：

1. 管理列表不需要选中态。
2. 本地上传直接调起本地文件选择器，不再弹二次弹窗。
3. 引用情况当前只区分：
   - 已引用
   - 未引用
4. 已引用模型不可删除。

#### 7.3.2 云端模型抽屉

入口：

1. 模型管理页中的“从云端添加”

查询条件：

1. 模型名称或版本编号
2. 场景类型

展示结构：

1. 按模型名称聚合
2. 每个模型下展示多个版本

版本项字段：

1. 版本编号
2. 训练完成时间
3. 下载状态或下载按钮

规则：

1. 版本行在同一行内撑满展示
2. 已下载版本展示“已下载”
3. 未下载版本展示“下载到本地”

#### 7.3.3 模型选择器

使用场景：

1. 图像处理实例选择分类模型
2. 图像检测实例选择检测模型

规则：

1. 只展示本地模型
2. 仅保留查询、选中、确认选择能力
3. 不在选择器中混入上传、下载、删除等管理行为

#### 7.3.4 场景类型口径

当前模型场景类型支持：

1. 尺寸
2. 分类
3. 缺陷检测

## 8. 相机管理

### 8.1 需求说明

相机需要从检测工具的临时配置中抽离出来，成为客户端独立资源，并通过参数组承载成像参数配置，以支持检测工具和采图工具复用。

### 8.2 方案描述

本期将相机管理拆分为两层：

1. 相机管理列表
2. 参数组管理

其中：

1. 相机列表负责资源发现、命名、删除和进入参数组管理。
2. 参数组管理负责维护每个相机下的成像参数方案。
3. 检测工具和采图工具仅消费相机与参数组，不直接编辑相机底层参数。

### 8.3 详细设计

#### 8.3.1 相机管理列表

页面结构：

1. 查询区：
   - 关键词
   - 厂商
2. 操作区：
   - 刷新
   - 添加相机
3. 列表字段：
   - 相机名称
   - 相机ID
   - 厂商
   - 型号
   - 状态
   - 操作

行操作：

1. 编辑名称
2. 参数组管理
3. 删除

规则：

1. 相机列表不再提供预览操作。
2. 参数组管理只有相机状态为 `空闲` 时可点击。
3. 已被引用的相机不可删除。

#### 8.3.2 添加相机

交互形式：

1. 弹窗

页面结构：

1. 搜索关键词
2. 搜索网络设备
3. 结果列表
4. 确认添加

规则：

1. 已添加相机直接过滤不显示。
2. 离线相机不出现在搜索结果中。
3. 支持多选添加。
4. 无可添加设备时展示空态。

#### 8.3.3 编辑名称

规则：

1. 支持自定义相机名称
2. 若名称为空，则回退到默认命名规则

#### 8.3.4 参数组管理

交互形式：

1. 弹窗

页面结构：

1. 左侧：参数组列表
2. 右侧上方：预览调试区
3. 右侧下方：参数详情

规则：

1. 页面标题统一为 `参数组管理`
2. 不展示相机厂商型号等额外说明文字
3. 左侧增加小标题 `参数组列表`
4. 新增参数组时直接插入列表并切换到右侧编辑
5. 删除入口位于参数组卡片内
6. 参数实时保存，不保留底部保存按钮
7. 参数详情区采用更紧凑的多列布局
8. 预览区按参数组图像宽高展示正确比例
9. 参数区为独立滚动区，避免整窗滚动抖动

## 9. 存储管理

### 9.1 需求说明

当前客户端需要显式表达存储风险，并允许用户主动管理历史数据，避免自动清理带来的数据追溯风险和现场认知偏差。

### 9.2 方案描述

本期将存储管理放入设置页，并采用“概览卡片 + 阈值设置 + 分类型清理”的方案：

1. 顶部用提醒条表达低于阈值的状态
2. 设置页内展示空间概览
3. 通过对应卡片内的清理入口，进入清理流程
4. 不再保留模型文件清理能力

### 9.3 详细设计

#### 9.3.1 设置页结构

设置页仅保留两个子页签：

1. 客户端信息
2. 存储管理

其中客户端版本已并入客户端信息，不再单独保留“软件信息”页签。

#### 9.3.2 存储管理页面

展示卡片：

1. 剩余可用空间
2. 检测记录图片
3. 采图记录图片

下方区域：

1. 阈值设置
   - 提醒阈值
   - 阻断阈值
   - 保存设置

规则：

1. 保存按钮位于两项阈值输入框右侧。
2. 模型文件卡片和模型清理功能已移除。

#### 9.3.3 提醒与阻断

顶部提醒规则：

1. 低于提醒阈值时，显示提醒条
2. 低于阻断阈值时，显示阻断提醒条
3. 提醒条附带 `去清理` 快捷按钮

阻断对象：

1. 新发起检测任务
2. 新发起采图任务
3. 本地模型上传
4. 云端模型下载

保留能力：

1. 查看历史记录
2. 进入设置调整阈值
3. 手动清理历史数据

#### 9.3.4 清理流程

清理入口：

1. 放在对应空间卡片内部

当前支持清理对象：

1. 检测记录相关数据
2. 采图记录相关数据

时间选择：

1. 一年前
2. 半年前
3. 三个月前
4. 一个月前
5. 自定义时间范围

交互顺序：

1. 先提示用户清理前导出备份
2. 再选择清理时间
3. 再执行确认

确认页展示原则：

1. 不再展示复杂清理规则说明
2. 不再展示“将清理的数据范围”
3. 不再展示“预计释放空间”
4. 仅保留清理对象、时间条件和受影响数据量

## 10. 开发测试评审建议

### 10.1 开发评审重点

1. 多客户端管理的数据口径是否与当前平台端/客户端页面一致
2. 检测工具的三步配置结构是否能覆盖现阶段业务
3. 检测运行和检测记录的三层结果结构是否统一
4. 模型管理与模型选择是否已完成职责拆分
5. 相机管理与参数组管理的边界是否清晰
6. 存储提醒、阻断、清理链路是否完整

### 10.2 测试评审重点

1. 首次绑定、已绑定直入、解绑回登录、额度不足、离线登录的行为是否正确
2. 检测工具配置链路是否完整，尤其是图像获取、ROI 绘制、模型选择
3. 检测运行页是否正确展示总结果、图像结果、子图结果
4. 检测记录列表、详情、结果查看、导出是否与三层结构一致
5. 相机管理、参数组管理的禁用与删除规则是否正确
6. 存储提醒、阻断提示、清理确认与清理后数据变化是否正确

### 10.3 当前不纳入完整验收

1. 采图记录完整页面
2. 采图运行页完整内容
3. 平台端除客户端管理外的其他业务模块
4. 真实后端接口与真实设备接入能力
