# product-v1_4-dev

JetCheck 1.4 开发工作区，当前主题是“尺寸检测场景支持”。

本工作区用于承接 JetCheck 1.4 版本开发。当前内容以 `product-v1_3-dev` 为基线复制而来，1.3 继续作为已评审基线保留，不直接在其上推进 1.4 需求。

## 当前状态（2026-04-09）

1. 路线已确认：独立本地尺寸工具 + JetCheck 融合。
2. 职责已确认：尺寸工具负责编辑/验证/发布，JetCheck 客户端负责正式运行与记录。
3. 核心框架已形成：标定、ROI、测量项、验证、发布、运行前校验、记录追溯。
4. 术语已初步收口：当前交互文档优先采用“ROI / 尺寸方案 / 发布版本 / 标定记录”。
5. 一期策略已收敛：保守推进，优先最小完整闭环。
6. 材料已收敛：本工作区仅保留 1.4 主线文档、demo、样例图、参考 PDF。

## 当前术语约定

1. `尺寸工程`
   - 编辑态容器，承载标定、ROI、测量项、验证数据。
2. `尺寸方案`
   - 面向客户端识别和引用的逻辑对象。
3. `发布版本`
   - 某次冻结后的可运行产物，客户端实际消费它。
4. `标定配置`
   - 某工程下的标定资源定义，约束适用相机/参数组/分辨率等条件。
5. `标定记录`
   - 某次具体标定的结果快照，系统内部保留用于追溯。
6. `ROI`
   - 当前交互文档中用于指代图像上的定位与操作区域。
