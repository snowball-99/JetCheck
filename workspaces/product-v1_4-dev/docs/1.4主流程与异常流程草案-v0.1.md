# JetCheck 1.4 主流程与异常流程草案 v0.1

本文档用于定义 1.4 尺寸检测方向的端到端流程。

当前重点不是页面细节，而是回答下面几个问题：

1. 用户从哪里开始配置尺寸工程
2. 标定、找正、测量项和验证是如何串起来的
3. 发布后 JetCheck 客户端如何消费尺寸方案并正式运行
4. 当现场条件发生变化时，系统如何自动恢复、引导重标定或直接阻断

## 1. 当前流程前提

本稿默认采用以下前提：

1. 尺寸工具负责编辑态、验证态、发布态
2. JetCheck 客户端负责正式运行态、记录态
3. 标定与尺寸方案解耦
4. 找正属于尺寸方案能力
5. 对现场变化采用保守策略
   - 能自动恢复的自动恢复
   - 需要重标定的引导重标定
   - 超出允许范围的直接阻断

## 2. 参与角色

建议先按下面 3 类角色理解。

### 2.1 质量工程师 / 工艺工程师

主要负责：

1. 创建尺寸工程
2. 标定
3. 找正配置
4. 测量项配置
5. 验证
6. 发布尺寸方案

### 2.2 一线操作员

主要负责：

1. 在 JetCheck 客户端选择并运行已发布尺寸方案
2. 查看结果
3. 在必要时执行受控的验证或求助维护人员

### 2.3 维护 / 调机人员

主要负责：

1. 现场相机条件检查
2. 重标定
3. 安装异常恢复

一期可以不把这类角色独立做成权限系统，但流程上建议保留这个分工。

## 3. 主流程总览

当前建议保留 4 条主流程：

1. 新建尺寸工程
2. 标定、找正与测量配置
3. 验证与发布
4. 客户端运行与记录

## 4. 主流程 1：新建尺寸工程

### 4.1 流程目标

建立一个可持续编辑、验证和发布的尺寸工程。

### 4.2 建议步骤

1. 用户启动尺寸工具
2. 选择“新建工程”
3. 填写工程基础信息
   - 工程名称
   - 适用产品 / 型号
   - 适用工位
   - 备注
4. 工具创建工程草稿
5. 工具初始化默认对象
   - 标定配置槽位
   - 找正设置槽位
   - 测量项列表
   - 验证数据集列表
6. 用户导入样本图或参考图

### 4.3 流程产物

输出一个“尺寸工程草稿”。

### 4.4 当前建议

1. 一期建议一个工程对应一个主尺寸方案方向
2. 不急着支持一个工程里分很多差异很大的方案分支

## 5. 主流程 2：标定、找正与测量配置

这条主流程建议再拆成 3 段理解：

1. 标定
2. 找正配置
3. 测量项配置

### 5.1 2A 标定流程

#### 流程目标

建立当前工程可用的像素尺寸换算关系。

#### 建议步骤

1. 用户在工程中进入“标定”
2. 选择标准标定板类型
3. 导入或采集标定图像
4. 系统调用内置标定算子自动识别
5. 系统输出标定结果摘要
   - 是否识别成功
   - 质量是否可接受
   - 是否可保存
6. 用户保存到当前标定配置，并生成一条标定记录
7. 用户填写或确认适用条件
   - 相机
   - 参数组
   - 分辨率
   - 可选工位条件

#### 流程产物

1. 标定配置
2. 标定记录

### 5.2 2B 找正配置流程

#### 流程目标

定义当前图像里如何找到参考位置，并把测量逻辑映射到实际图像。

#### 建议步骤

1. 用户选择参考图
2. 用户定义搜索 ROI
3. 用户定义参考特征
4. 系统生成找正规则
5. 用户查看找正后的叠加效果
6. 用户确认允许偏差范围和失败处理策略

#### 流程产物

1. 找正设置
2. 搜索 ROI

### 5.3 2C 测量项配置流程

#### 流程目标

定义真正要输出的尺寸结果。

#### 建议步骤

1. 用户新增测量项
2. 选择测量类型
3. 选择关联区域或几何基元
4. 系统生成可视化叠加元素
5. 用户设置输出单位、精度和可选公差
6. 用户保存测量项

#### 流程产物

1. 测量 ROI
2. 测量项

### 5.4 当前建议

1. 一期先支持少量高频测量项
2. 一期先支持一种主找正方式
3. 不建议一开始做开放式算法节点编排

## 6. 主流程 3：验证与发布

### 6.1 流程目标

确认尺寸方案在样本图上可用，并生成 JetCheck 客户端可消费的发布产物。

### 6.2 建议步骤

1. 用户选择验证数据集
2. 用户发起单次验证或批量验证
3. 系统执行：
   - 使用当前工程草稿
   - 使用当前标定记录
   - 执行找正
   - 执行测量项输出
4. 系统展示验证结果
   - 测量值
   - 叠加结果
   - 可选超差结果
   - 异常图
5. 用户确认是否满足发布条件
6. 若满足条件，执行发布
7. 系统生成：
   - 尺寸方案
   - 发布版本
   - 依赖的标定配置引用关系
8. 系统把发布版本发布到 JetCheck 可消费目录

### 6.3 流程产物

1. 验证任务记录
2. 尺寸方案
3. 发布版本

### 6.4 当前建议

1. 一期至少支持单次验证
2. 轻量批量验证强烈建议纳入一期
3. 发布前应至少校验：
   - 有标定
   - 有找正
   - 有测量项
   - 有基本验证结果

## 7. 主流程 4：客户端运行与记录

### 7.1 流程目标

让 JetCheck 客户端正式运行尺寸方案，并把结果纳入现有检测记录体系。

### 7.2 建议步骤

1. JetCheck 客户端同步可用尺寸方案目录
2. 用户在 JetCheck 运行工具中引用某个尺寸方案
3. 正式运行前，客户端执行运行前校验
4. 若校验通过：
   - 读取发布版本
   - 读取发布版本依赖的标定配置
   - 获取当前生效的标定记录
5. JetCheck 取图
6. JetCheck 执行找正
7. JetCheck 执行测量项
8. JetCheck 输出测量值
9. JetCheck 将尺寸结果写入现有检测记录体系

### 7.3 记录里建议保留的信息

1. 尺寸方案 ID
2. 发布版本 ID
3. 标定配置 ID
4. 实际使用的标定记录 ID
5. 相机 / 参数组快照
6. 测量项结果列表
7. 可选超差结果
8. 结果叠加图层摘要

### 7.4 当前建议

1. 客户端实际运行使用的是发布版本
2. 运行时落的是“发布版本 + 标定记录”的实际快照

## 8. 异常流程 1：运行前发现条件不匹配

### 8.1 触发时机

JetCheck 正式运行前。

### 8.2 建议校验项

1. 相机是否匹配
2. 参数组是否匹配
3. 分辨率是否匹配
4. 当前条件是否落在标定适用范围内
5. 当前条件是否落在发布版本允许范围内

### 8.3 处理策略

当前建议把结果分成 3 档：

1. 可自动恢复
   - 允许继续运行
   - 记录本次恢复情况
2. 需重新标定
   - 阻断正式运行
   - 引导进入重标定
3. 超出安装允许范围
   - 直接阻断
   - 提示先调整相机安装

### 8.4 当前建议

一期默认不要只告警不拦截。

原因：

1. 尺寸检测里运行条件失真，结果本身可能不可信
2. 保守一点更符合当前阶段

## 9. 异常流程 2：需重新标定，但不应重发发布版本

### 9.1 流程目标

解决“标定失效”但尺寸方案逻辑仍有效的情况。

### 9.2 建议步骤

1. 客户端或维护人员发现当前标定不再适用
2. 发起重标定流程
3. 系统在原标定配置下生成新的标定记录
4. 新标定记录通过基础校验后，切换为当前生效记录
5. 发布版本保持不变
6. 客户端后续运行自动使用新标定记录

### 9.3 当前建议

这是 1.4 非常重要的一条产品原则：

- 重新标定默认只更新标定记录，不要求重新发版

## 10. 异常流程 3：安装异常

### 10.1 典型场景

1. 相机角度变化明显
2. 整体倾斜或偏移过大
3. 透视关系明显失真
4. 现场已经超出尺寸方案和标定的前提

### 10.2 建议处理方式

1. 阻断运行
2. 明确提示“当前安装条件超出允许范围”
3. 引导先调整安装
4. 调整后再做验证或重标定

### 10.3 当前建议

一期宁可保守阻断，也不建议用“可能还能跑”的模糊策略放过。

## 11. 异常流程 4：尺寸方案逻辑变化

### 11.1 典型场景

1. 测量项变了
2. 找正规则变了
3. 尺寸方案结构本身变了

### 11.2 建议处理方式

1. 回到尺寸工具编辑工程
2. 重新验证
3. 重新发布新版本
4. 客户端更新方案引用

### 11.3 和重标定的区别

这里需要特别区分：

1. 标定变化
   - 默认不重发发布版本
2. 尺寸方案逻辑变化
   - 必须发新发布版本

## 12. 当前最关键的流程原则

基于这版草案，我建议先统一接受下面这 6 条原则：

1. 编辑、验证、发布都发生在尺寸工具
2. 正式运行和记录都发生在 JetCheck 客户端
3. 标定失效不等于尺寸方案失效
4. 找正优先解决工件位置变化
5. 运行前必须有条件校验
6. 条件超出允许范围时优先阻断

## 13. 当前我最建议下一步继续做什么

基于对象和流程已经落地，下一步最建议继续做这两件事：

1. 输出“1.4 一期白名单能力清单”
2. 输出“JetCheck 客户端运行前校验规则草案”

如果只能先做一份，我建议优先写运行前校验规则草案。
